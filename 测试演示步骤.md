## 🚀 快速启动

### 方法一：使用启动脚本（推荐）

1. 双击运行 `start-project.bat` 文件
2. 等待两个服务器启动（约10秒）
3. 浏览器会自动打开应用

### 方法二：手动启动

**打开第一个命令行窗口（后端）：**
```bash
cd Lab-17/Lab-17/nodemon-lab
npm run dev
```

**打开第二个命令行窗口（前端）：**
```bash
cd Lab-19/Lab-19/my-portfolio
npm run dev
```

**在浏览器中访问：**
```
http://localhost:5174
```

## ❓ 常见问题

### Q1: 输入框中看不到文字怎么办？

**A:** 这个问题已经修复！所有输入框现在都有：
- 白色背景 (`bg-white`)
- 深灰色文字 (`text-gray-900`)
- 浅灰色边框 (`border-gray-300`)

如果仍然看不到，请：
1. 刷新浏览器（Ctrl+F5 强制刷新）
2. 清除浏览器缓存
3. 检查是否使用了浏览器扩展修改了页面样式

### Q2: 前端无法连接后端怎么办？

**A:** 检查以下几点：
1. 确保后端服务器正在运行（http://localhost:3000）
2. 确保前端服务器正在运行（http://localhost:5174）
3. 检查浏览器控制台是否有 CORS 错误
4. 检查 `.env` 文件中的 `VITE_API_URL` 是否正确

### Q3: 登录后立即退出怎么办？

**A:** 可能的原因：
1. JWT token 无效
2. localStorage 被清除

解决方法：
1. 打开浏览器开发者工具（F12）
2. 进入 Application > Local Storage
3. 删除所有 localStorage 数据
4. 重新登录

### Q4: 数据库没有数据怎么办？

**A:** 运行数据库种子脚本：
```bash
cd Lab-17/Lab-17/nodemon-lab
node seedData.js
```

这会创建：
- 管理员账户（admin@test.com / password123）
- 3个示例项目
- 3篇示例博客文章

### Q5: 端口被占用怎么办？

**A:** 如果看到 "Port 5173 is in use"：
- Vite 会自动使用下一个可用端口（如 5174）
- 查看终端输出确认实际端口号
- 使用显示的端口号访问应用

### Q6: 如何重置所有数据？

**A:** 
1. 停止所有服务器
2. 重新运行种子脚本：
```bash
cd Lab-17/Lab-17/nodemon-lab
node seedData.js
```
3. 清除浏览器 localStorage
4. 重新启动服务器
