### 详细步骤

## 第一步：注册 GitHub 并上传代码

### 1.1 注册 GitHub 账号
1. 访问 https://github.com/signup
2. 填写邮箱、密码、用户名
3. 验证邮箱
4. 完成注册

**提示**：
- 使用常用邮箱（QQ邮箱、163邮箱都可以）
- 用户名建议用英文，会显示在项目地址中

### 1.2 创建仓库
1. 登录 GitHub
2. 点击右上角 "+" → "New repository"
3. 填写信息：
   - **Repository name**: `fullstack-portfolio`
   - **Description**: `Full-stack portfolio project`
   - **选择**: Public（公开）
   - ⚠️ **不要勾选**任何选项（README、.gitignore、license）
4. 点击 "Create repository"
5. **保持这个页面打开**，后面会用到

### 1.3 创建 Personal Access Token（重要！）

GitHub 不支持密码推送代码，需要创建 Token：

1. 点击右上角头像 → Settings
2. 左侧菜单最下方 → Developer settings
3. Personal access tokens → Tokens (classic)
4. Generate new token → Generate new token (classic)
5. 填写信息：
   - **Note**: `deploy-token`
   - **Expiration**: `No expiration`（永不过期）
   - **勾选**: `repo`（勾选后下面会自动全选）
6. 滚动到底部，点击 "Generate token"
7. **复制生成的 Token**（以 `ghp_` 开头，很长一串）
8. ⚠️ **重要**：把 Token 保存到记事本，只会显示一次！

### 1.4 推送代码到 GitHub

#### 方法一：使用 Git 命令行（推荐）

1. 打开项目文件夹
2. 右键 → "Git Bash Here"（如果没有，需要先安装 Git）
3. 依次执行以下命令：

```bash
# 初始化仓库
git init

# 配置用户信息（替换成你的）
git config --global user.email "你的邮箱@example.com"
git config --global user.name "你的用户名"

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit: Full-stack portfolio project"

# 重命名分支
git branch -M main

# 添加远程仓库（替换成你的仓库地址）
git remote add origin https://github.com/你的用户名/fullstack-portfolio.git

# 推送代码
git push -u origin main
```

4. 当要求输入用户名和密码时：
   - **Username**: 你的 GitHub 用户名
   - **Password**: 粘贴刚才复制的 Token（不是 GitHub 密码！）

5. 等待上传完成（可能需要几分钟）

#### 方法二：使用 GitHub Desktop（更简单）

1. 下载 GitHub Desktop：https://desktop.github.com
2. 安装并登录 GitHub 账号
3. File → Add Local Repository → 选择项目文件夹
4. 点击 "Publish repository"
5. 确认信息，点击 "Publish"

### 1.5 验证上传成功

1. 回到 GitHub 网站
2. 刷新仓库页面
3. 应该能看到所有文件

## 第二步：部署后端到 Render

### 2.1 注册 Render
1. 访问 https://render.com
2. 点击 "Get Started"
3. 选择 "Continue with GitHub"（用 GitHub 登录）
4. 授权 Render 访问你的 GitHub

### 2.2 创建 Web Service
1. 点击 "New +" → "Web Service"
2. 点击 "Connect account"（如果需要）
3. 找到 `fullstack-portfolio` 仓库，点击 "Connect"

### 2.3 配置服务

填写以下信息：

```
Name: portfolio-backend
（或任意名称，会成为网址的一部分）

Region: Singapore
（选择离你最近的区域，中国用户选 Singapore）

Branch: main

Root Directory: Lab-17/Lab-17/nodemon-lab
（⚠️ 重要！必须填写这个路径）

Runtime: Node

Build Command: npm install

Start Command: npm start

Instance Type: Free
（选择免费套餐）
```

### 2.4 添加环境变量

滚动到 "Environment Variables" 部分，点击 "Add Environment Variable"，添加以下变量：

**变量 1:**
```
Key: MONGODB_URI
Value: mongodb+srv://ahsanshehzad_db_blogUser:uAL9CkQ1JVPQJjJB@blogplatformcluster.2cue0xo.mongodb.net/?retryWrites=true&w=majority&appName=BlogPlatformCluster
```

**变量 2:**
```

Value: your_jwt_secret_key1122
```

**变量 3:**

**变量 4:**

### 2.5 部署
1. 点击 "Create Web Service"
2. 等待 3-5 分钟（会显示部署日志）
3. 看到 "Live" 绿色标志表示成功
4. **复制你的后端 URL**（类似：`https://portfolio-backend-xxxx.onrender.com`）
5. 保存这个 URL，下一步会用到

## 第三步：部署前端到 Vercel

### 3.1 注册 Vercel
1. 访问 https://vercel.com
2. 点击 "Sign Up"
3. 选择 "Continue with GitHub"（用 GitHub 登录）
4. 授权 Vercel 访问你的 GitHub

### 3.2 导入项目
1. 点击 "Add New..." → "Project"
2. 找到 `fullstack-portfolio` 仓库
3. 点击 "Import"

### 3.3 配置项目

填写以下信息：

```
Framework Preset: Vite
（应该会自动检测）

Root Directory: Lab-19/Lab-19/my-portfolio
（点击 Edit，填写这个路径）

Build Command: npm run build
（应该会自动填写）

Output Directory: dist
（应该会自动填写）
```

### 3.4 添加环境变量

点击 "Environment Variables"，添加：

```
Key: VITE_API_URL
Value: 你的Render后端URL/api
```

**例如**：
```
https://portfolio-backend-xxxx.onrender.com/api
```

⚠️ **注意**：
- 使用第二步复制的 Render URL
- 末尾必须加 `/api`
- 不要有多余的空格

### 3.5 部署
1. 点击 "Deploy"
2. 等待 2-3 分钟
3. 看到庆祝动画表示成功
4. 点击 "Visit" 或复制生成的 URL

## ⚠️ 常见问题

### Q1: Git 命令不识别？
**A**: 需要先安装 Git
- 下载：https://git-scm.com/download/win
- 安装时全部选默认选项
- 安装完重启命令行

### Q2: 推送代码时密码错误？
**A**: 不要输入 GitHub 密码，要输入 Token！
- Token 是以 `ghp_` 开头的长字符串
- 在第一步 1.3 创建的

### Q3: Render 部署失败？
**A**: 检查以下几点：
- Root Directory 是否填写：`Lab-17/Lab-17/nodemon-lab`
- 环境变量是否都添加了（5个）
- Build Command 是否是：`npm install`
- Start Command 是否是：`npm start`

### Q4: Vercel 部署成功但网站打不开？
**A**: 检查：
- Root Directory 是否填写：`Lab-19/Lab-19/my-portfolio`
- VITE_API_URL 是否正确（要加 /api）
- 后端是否已成功部署

### Q5: 网站能打开但无法登录？
**A**: 
- 检查 Vercel 的 VITE_API_URL 是否正确
- 检查后端 Render 是否在运行（访问后端 URL 看是否有响应）
- 等待 30 秒（Render 免费版首次访问需要唤醒）

### Q6: Render 显示 "Service Unavailable"？
**A**: 
- 免费版 15 分钟无活动会休眠
- 首次访问需要 30-60 秒唤醒
- 刷新页面等待即可
