## 📤 部署到 GitHub + Render + Vercel

### 第一步：上传代码到 GitHub

#### 1. 创建 GitHub 仓库
1. 访问 https://github.com/new
2. 仓库名称：`fullstack-portfolio`（或自定义）
3. 选择 Public（公开）
4. **不要勾选**任何选项
5. 点击 "Create repository"

#### 2. 推送代码
在项目根目录执行：

```bash
git init
git add .
git commit -m "Initial commit: Full-stack portfolio project"
git branch -M main
git remote add origin https://github.com/你的用户名/fullstack-portfolio.git
git push -u origin main
```

**注意**：GitHub 需要 Personal Access Token，不支持密码登录。

**创建 Token：**
1. 访问 https://github.com/settings/tokens/new
2. Note: `deploy-token`
3. Expiration: `No expiration`
4. 勾选: `repo`
5. 点击 "Generate token"
6. 复制 Token（以 ghp_ 开头）
7. 推送时用 Token 代替密码

```
Runtime: Node
Build Command: npm install
Start Command: npm start
Instance Type: Free
```

### 第三步：部署前端到 Vercel

#### 1. 注册/登录 Vercel
- 访问 https://vercel.com
- 使用 GitHub 账号登录（推荐）

#### 2. 导入项目
1. 点击 "Add New..." → "Project"
2. 选择 `fullstack-portfolio` 仓库
3. 点击 "Import"

#### 3. 配置设置
```
Framework Preset: Vite
Root Directory: Lab-19/Lab-19/my-portfolio
Build Command: npm run build
Output Directory: dist
```

#### 4. 环境变量
添加环境变量：

```
VITE_API_URL=你的Render后端URL/api
```

例如：`https://portfolio-backend-xxxx.onrender.com/api`

#### 5. 部署
1. 点击 "Deploy"
2. 等待 2-3 分钟
3. 访问生成的 URL

## 🔧 常见问题

### Q1: 推送代码时要求密码但密码不对？
**A**: GitHub 不再支持密码，需要使用 Personal Access Token。参见上面的"创建 Token"部分。

### Q2: Render 部署失败？
**A**: 检查：
- Root Directory 是否正确
- 环境变量是否都已添加
- Build Command 和 Start Command 是否正确

### Q3: 前端无法连接后端？
**A**: 检查：
- Vercel 的 VITE_API_URL 是否正确
- 后端 URL 末尾要加 `/api`
- 后端是否已成功部署

### Q4: 本地运行报错？
**A**: 
1. 删除 node_modules 文件夹
2. 运行 `npm install`
3. 检查 Node.js 版本（需要 14+）
